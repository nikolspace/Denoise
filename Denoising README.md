# **Denoising-GB: Hybrid Microstructural Noise Cleaning Framework**

This repository provides the complete programmatic methodology and deep learning framework for **Microstructural Denoising** using a hybrid synthetic-real methodology.

## **Methodology**

The dataset is generated by superimposing real-world imaging noise (scratches, pits, and sensor noise) onto clean polycrystalline templates generated via the polySim library.

1. **Template Generation:** Clean templates are produced via polySim.  
2. **Grain Isolation:** Each grain is isolated using bounding-box detection (phasepick).  
3. **Real-Noise Integration:** A real noise crop is randomly selected, rotated, and resized to fit the isolated grain.  
4. **Otsu Pre-processing:** The resulting noisy micrograph is thresholded using Otsu's method to create the "Noisy Input" for the model.

## **Installation**

pip install polySim numpy opencv-python matplotlib scikit-image tensorflow keras sklearn numba

## **1\. Dataset Generation**

Requires a folder of "noise crops" (small grayscale images containing only microstructural artifacts).

python denoise\_data\_generation.py \--noise\_dir ./real\_noise\_crops \--count 1000 \--output ./dataset

## **2\. Training and Testing**

The framework implements a **4-Layer U-Net** optimized with a hybrid **Dice \+ Binary Cross-Entropy loss** and evaluated using **Intersection over Union (IoU)**.

### **How to Run:**

python train\_and\_test\_denoising.py \--data ./dataset \--epochs 100

### **Key Features:**

* **Resume Capability:** Automatically resumes training from the last saved checkpoint if the process is interrupted.  
* **Metric-Driven Training:** Uses IoU to monitor real geometric overlap between predicted boundaries and ground truth.  
* **Hybrid Loss:** Dice loss ensures connectivity while BCE ensures pixel-wise accuracy.

## **Dataset Structure**

* noisy\_input/: Binary (Otsu-thresholded) noisy micrographs.  
* ground\_truth\_template/: Clean, fully connected polycrystalline grain boundary skeletons.
